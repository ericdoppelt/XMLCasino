LobbyView XML Page


The LobbyView XML file contains meta tags for starting Dimensions (Width and Height). Similar to the GameView XML file, it also contains Stylesheets and Languages which drive the creation of the SettingsBar on top. There is also an IconProperties bundle which dictates the icons in the Settings bar, as well as an ErrorStylehseet reference.


The next section contains two tags which drive the IconBundles contained within the Validation view (FilesStatusBundle and FilesIconBundle). These customize the appearance of the validation view and are validated with enumerated types internal to the schema.


The final section contains a series of Bundle tags which encode information that creates Icons seen in the LobbyView. Each Icon contains a Name, an Icon, and a sequence of Files (Deck, Game, Hands, Players, and View). All of the icons are validated by enumerated types in the schema, and the file names are validated in the validation view sequence. Each bundle must include a Name and an Icon tag, and then contains an arbitrary number of files. The rest of file selection happens in the Validation loop.
